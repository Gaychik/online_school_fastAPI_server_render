# 📦 Модульная структура секций

## Описание

Все секции лендинга разделены на отдельные HTML файлы для удобной поддержки и переиспользования.

---

## 📁 Структура файлов

### **templates/sections/**

Каждая секция - отдельный файл:

```
sections/
├── section_1_banner.html          # Баннер с таймером
├── section_2_hero.html            # Главный экран (Hero)
├── section_3_subjects.html        # Предметы
├── section_4_benefits.html        # Преимущества
├── section_5_program.html         # Описание программы
├── section_6_calendar.html        # Календарь занятий
├── section_7_info.html            # Дополнительная информация
├── section_8_reviews.html         # Отзывы (Swiper)
├── section_9_packages.html        # Пакеты и цены
├── section_10_consultation.html   # Форма консультации
├── section_11_faq.html            # FAQ (аккордеон)
├── section_12_license.html        # Лицензия
└── section_13_final_cta.html      # Финальная форма
```

---

## 🔧 Как использовать

### Подключение в index.html:

```html
{% extends "base.html" %}

{% block content %}

<!-- Banner -->
{% include 'sections/section_1_banner.html' %}

<!-- Hero Section -->
{% include 'sections/section_2_hero.html' %}

<!-- И так далее... -->

{% endblock %}
```

## ✨ Преимущества модульной структуры

### 1. **Удобство поддержки**
- Каждая секция в отдельном файле
- Легко найти и исправить ошибку
- Понятная структура

### 2. **Переиспользование**
- Можно комбинировать секции
- Легко добавить новую секцию
- Можно отключить ненужные секции

### 3. **Командная разработка**
- Разные разработчики могут работать с разными секциями
- Меньше конфликтов в Git
- Четкое разделение ответственности

### 4. **Масштабируемость**
- Легко добавить A/B тесты (просто подменить секцию)
- Можно создавать вариации страниц
- Удобно для экспериментов

## 📝 Примеры использования

### Добавить новую секцию:

1. Создайте файл `templates/sections/section_14_new.html`
2. Напишите HTML код секции
3. Добавьте в `index.html`:
   ```html
   {% include 'sections/section_14_new.html' %}
   ```

### Отключить секцию:

Закомментируйте или удалите строку в `index.html`:

```html
<!-- {% include 'sections/section_7_info.html' %} -->
```

### Изменить порядок секций:

Просто поменяйте местами include в `index.html`:

```html
{% include 'sections/section_3_subjects.html' %}
{% include 'sections/section_2_hero.html' %}
<!-- Теперь Subjects идет перед Hero -->
```

## 💡 Советы

### Именование файлов:

- Используйте префикс `section_X_` для порядка
- Давайте понятные имена (`hero`, `reviews`, `prices`)
- Следуйте единому стилю (нижний регистр, подчеркивания)

### Организация кода:

- Одна секция = один файл
- Избегайте дублирования кода
- Выносите повторяющиеся элементы в partials

### Оптимизация:

- Не создайте слишком большие секции (>500 строк)
- Разбивайте сложные секции на подсекции
- Используйте комментарии для навигации

## 🔄 Миграция со старой структуры

Раньше все секции были в `main_sections.html`. Теперь:

```
main_sections.html (удален)
↓
section_2_hero.html
section_3_subjects.html
section_4_benefits.html
...и так далее
```

Если у вас были кастомные изменения в `main_sections.html`, перенесите их в соответствующие файлы секций.
